## 4) 우선 개선 백로그 (상용화 직결)

### P0 (이번 스프린트)
1. **i18n 리소스 파일 분리**
   - 하드코딩 문자열을 locale 파일로 이동
   - ko/en 외 확장 준비
2. **행정동 alias 사전 외부화**
   - 코드 상수 → 데이터 파일(JSON/TS)로 분리
   - 운영 중 핫픽스 가능 구조
3. **검색 품질 메트릭 추가**
   - `search_success_rate`, `no_result_rate`, `fallback_hit_rate`

### P1 (다음 스프린트)
1. **키보드 네비게이션 완성**
   - 결과 리스트 화살표/Enter 선택 강화
2. **지도 포커스 애니메이션 통일**
   - 선택 시 부드러운 pan/zoom 규칙 고정
3. **접근성 점검 (A11y)**
   - ARIA, 포커스 순서, 명도 대비 재검증

### P2 (확장)
1. **멀티 해상도 H3(7/8/9) UX**
2. **지역별 검색 사전(국가별 transliteration)**
3. **검색 품질 대시보드 운영화**

## 5) 반복 검증 루프 (실행 규칙)

변경 1회마다 아래를 반복:
1. 프론트 빌드
2. 릴리즈 스모크
3. UI 시나리오 수동 체크
4. 실패 시 즉시 롤백/재수정 후 재실행

권장 명령:
- `npm --prefix tauri-shell run build`
- `./scripts/smoke_release.ps1`
- `./scripts/ui_smoke.ps1`

---

## 6) 수동 QA 시나리오 (최소)

1. `성수동` 검색 → 결과 리스트 표시
2. 결과 카드 클릭 → 지도 선택 동기화
3. 빈 검색어/짧은 검색어 → 안내 문구 확인
4. 무결과 검색어 → 무결과 + 추천 검색어 표시
5. 리포트 열기/닫기 → 포커스/상태 유지
6. 설정 모달 ESC/외부 클릭 닫기
